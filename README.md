# Лабораторная работа №1: «Основы программирования на высокоуровневых языках» (C++20/23)

## 📌 Техническое задание (ТЗ)

### Общие требования

1. **Проект на git** – весь код находится в репозитории с историей изменений.
2. **Интерактивный диалоговый интерфейс** – для проверки корректности программы.
3. **Корректное завершение** – без `exit`, `abort`, `std::terminate`, без утечек памяти.
4. **Логичная структура проекта** – каждый модуль в своих файлах, `main` и диалоговые функции – отдельно.
5. **Автосборка** – CMake (или qmake/Makefile).
6. **Статический анализ** – Clang Static Analyzer / SonarQube / VS Code Analysis. Все замечания исправлены или обоснованы.
7. **Динамический анализ** – Valgrind / Deleaker / VS Diagnostics. Утечки памяти отсутствуют.
8. **Качество кода** – разумная декомпозиция, DRY, корректное использование заголовочных файлов.
9. **Стандарт языка** – C++20 или C++23.
10. **Сдача в IDE** – с настроенным LSP (clangd / Visual Studio / CLion).

## 🔄 Порядок выполнения работы (инкрементально)

Каждый следующий пункт выполняется поверх предыдущего. Для опытных допускается выполнение всех пунктов сразу.

### 1. Реализация (C → C++ с new/delete)
- Реализовать функции по индивидуальному заданию на **C**.
- Простая диалоговая программа: `dialog_start_operation` – ввод значений, вызов функции, вывод результата.
- **Память:** заменить `malloc/calloc/realloc/free/strdup` на `new/delete`.

### 2. Исключения
- Механизм обработки ошибок – только **исключения** (`throw`, `try`/`catch`).
- Коды возврата не используются.

### 3. Перегрузки
- Добавить перегрузки реализованной функции (согласно заданию).
- Все перегрузки используют **то же имя**, что и основная функция.

### 4. Ссылки
- Передача параметров – через **ссылки (`&`)** или **константные ссылки (`const &`)**.
- Передача по указателю или по значению – только с обоснованием.

### 5. Ввод-вывод
- Полностью на **`<iostream>`** (`std::cin`, `std::cout`, `std::cerr`).
- Использование `printf`/`scanf` и `stdio.h` **недопустимо**.

## 🚀 Дополнительные задачи (для продвинутых)

### 6. Алгоритмы (`<algorithm>`)
- Замена всех циклов внутри функции на примитивы из `<algorithm>`:
  - `std::find` – поиск
  - `std::copy` – копирование массивов
  - `std::transform`, `std::sort`, `std::for_each` и др.

### 7. Ranges (`<ranges>`) – только для самых опытных
- Использование **библиотеки ranges** (аналог list comprehension в Python / stream в Java).
- Алгоритмы над множествами (constraints).
- Для нестандартных задач – `std::generator`.
